Overview

In this assignment, we implemented some core routines of a physically-based renderer using pathracing. We began by implementing ray generation and constructing the BVH algorithm to accelerate rendering efficiency of more complex geometries. Then, we constructed algorithms for direct and indirect illumination. Lastly, we implemented adaptive sampling to decrease sampling for pixels that have already converged.

Part 1

Walk through ray generation.

We implemented ray generation as follows:

First, we transformed normalized coordinates from image space to camera space. We did so by scaling the x and y coordinates in image space by the sensor width and height, respectively, with the x and y coordinates of the bottom left corner as the origin. The z coordinate remains -1 throughout as the sensor lies on the z = -1 plane.

Second, we found the parameters for generating a ray in camera space. That ray would have an origin of (0, 0, 0) in camera space and its direction vector would be its intersection with the sensor.

Third, we generated a ray in world space. The origin of the camera in world space is given by pos. We find the direction vector in world space by transforming the camera direction vector with the c2w camera-to-world rotation matrix. The world origin and normalized world direction of the camera constitute the final parameters for the generated ray.

Explain the triangle intersection algorithm you implemented.

First we checked if the ray intersected with the plane defined by the triangle. A plane is defined by the normal vector and a point on the plane. One of the points of the triangle was used for the point, and then the cross product of two of the vectors of the triangle was computed for the normal vector. Then, with the given plane and ray, we solved for the parameter t. As long as 0 <= t < inf. We have a valid intersection into the plane.

With the parameter t we calculate the exact intersection point. Now, we want to check if the intersection point is within the triangle. To do this, we calculate the barycentric coordinates of the intersection point relative to the triangle. If all coefficients are positive and sum to 1, we know that the point is in the triangle.

Explain the sphere intersection algorithm you implemented.

We implemented the test helper function to see if 1) if a ray intersects the sphere and 2) the intersection time(s) that occurs. We solved for the intersection between the ray and sphere (equations) through substitution—setting p=o+td, and used the quadratic formula to find roots denoting intersection times. If there were real roots, we recorded the min root in t1 and max in t2.

Using test, we could find the sphere intersection by first verifying the validity of both t values (0 <= ray.min_t <= t <= ray.max_t < inf). We took the min or closest t value that was valid to find the ray hit point, calculated as ray origin + t * ray direction. This allowed us to calculate the sphere normal for shading, defined per convention as the normalized value of the difference between the hit point and ray origin vectors.

Part 2

Walk through your BVH construction algorithm. Explain the heuristic you chose for picking the splitting point.

We implemented the BVH construction algorithm as follows:

First, we defined the base case. If the number of primitives in a node falls to or below the max leaf size, we construct a leaf node with no left or right children and return it.

Otherwise, we split the node. The split axis is selected such that it has the highest variation among primitive objects i.e. the greatest extent (difference between min and max). Then, we sorted the primitives in the node by the corresponding axis coordinates of their centroids.

The heuristic we chose for the splitting point was the average of centroids along an axis, which ensured an even distribution in the number of primitives allocated to the left and right children. Lastly, we recurse and call construct_bvh on the left and right children until we reach the leaf node base case.

Part 3

Walk through uniform hemisphere sampling.

For uniform hemisphere sampling, we calculate the hit point and the vector exiting the hit point (into the camera). Then, we sample for inbound vectors into the hit point with hemisphereSampler->get_sample().We then create a ray from hit_point + EPS_F and check if it intersects with anything. If it does, we use the reflection equation to calculate how much light the source contributes to the outbound ray. We take the sum of all the contributions and then divide by the number of samples.

Walk through light importance sampling.

For light importance sampling, we calculate the hit point and the vector exiting the hit point (into the camera). Then, we iterate over all the light sources and sample them either ns_area_light times (if it is an area light), or 1 time if it is a point light. We then get the inbound vector, distance to the light and the pdf of that light and use the reflection equation to calculate how much light is contributed to the outbound vector. For each light, we divide by the number of times we sampled the light.

Uniform sampling tends to give much more grainy results; there are cases where adjacent pixels should look roughly the same but do not due to the variance in the hemisphere sampling; the estimator can overestimate and underestimate the true amount of light reflected. Lighting importance sampling gets rid of this problem by sampling every light for each pixel, guaranteeing much more consistent results / a much less variable result in terms of light reflected.

Part 4

Walk through your implementation of the indirect lighting function.

We implement indirect lighting function through recursive radiance calculations. We only proceed in the case that further bounces should be simulated, i.e. r.depth > 0. L_out = one_bounce_radiance(r, isect) calculates the radiance returned from the first bounce. Then, we get further bounces by sampling a new direction for the ray based on the BSDF at the intersection point. A new ray next_ray is created with its origin at the intersection point and its direction set to the sampled direction transformed back to world space. We then decrement the depth of the ray to reflect that a bounce has been simulated. We recursively call at_least_one_bounce_radiance for the new ray, multiplying the returned radiance by the BSDF value, the cosine of the angle between the normal and the outgoing direction, and dividing by the PDF, to get the correct contribution from this path.

Part 5

Explain adaptive sampling. Walk through your implementation of the adaptive sampling.

Adaptive sampling is an algorithm that concentrates higher sampling on only the more difficult parts of an image, terminating at a lower sampling rate when the pixel has converged. To implement adaptive sampling, we modified the raytrace_pixel function from part 1 to account for the following:

Each pixel has a maximum of ns_aa rays, so that is set to be the loop termination condition. We used get_sample() to randomize a sample within the pixel, used offsetting to find that sample’s position relative to the given x and y coordinates, and lastly found its normalized position on the samplebuffer by dividing by the samplebuffer’s width and height for x and y respectively. We generated rays for each of those samples, and set their depth. We found the pixel value by adding together all the scene radiance from generated rays and dividing by the number of samples.

We accounted for adaptive sampling by performing illumination summation across samples for s1 and square of illumination summation for s2. When the number of samples reached a multiple of samplesPerBatch, we performed a convergence check in which we calculate the mean and standard deviation from s1 and s2, calculate I, and check if I <= max_tolerance * mean or we’ve reached the maximum ns_aa rays generated. If either of those conditions are true, we broke from the loop. If those were not true, we would continue sampling another samplesPerBatch times. The idea is, pixels that reach convergence earlier would not have to have as a high a sampling rate as those that converged later.
